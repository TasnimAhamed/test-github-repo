1. What is git?
    - Git is a version control system/tool.

/* Important Terms */
1. Working Directory : The folder where I am working.

2. Local Repository: When git track a local folder and track them and versionize, call it Local Repository.

3. Remote Repository


# Download and Install git: https://git-scm.com/downloads

/* Basic git Configuration */

Commads:

1. Check git Configuration : git config --list

2. Login/add username : git config --global user.name "YOUR_USERNAME"
  - Login/add email : git config --global user.name "YOUR_EMAIL"

/************** Userful Commands **************/

1. present working directory: pwd 
    - File list : ls
    - File list with hidden files: ls -a
    - remove file : rm "file_name"
    - force to remove : rm -f "file_name"
    - clear the termina: clear(git) / cls(powershell)

2. create a file : touch "file_name"
    - Example    : touch readme.md

3. check git status: git status 

4. make this working directory to a local repository: git init

5. Track a file : git add file_name"
    - Track all files : git add --all 
                      : git add .

6. Untrack/Unstage a file: git rm --cached "file_name" 

7. Get your track files: git commit -m "your_commit_message"

8. Get git history/log : git log
            get it in short: git log --oneline

9. Discard the change in working directory: git restore "file_name"

10. Revert a commit: git reset --hard "commit_id"

11. Get all logs: git reflog


/************** Branching in Git **************/

1. Create a new branch: git branch "branch_name"

2. Create a new branch and switch to the new branch: git checkout -b "branch_name"

3. Get branch list : git branch
                   : git branch --list

4. Switch to new branch: git switch "branch_name"

5. Merge the new branch: git merge "branch_name"

6. Delete a branch: git branch -d "branch_name"
